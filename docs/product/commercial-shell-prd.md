# 微信多开商用工作台 PRD

版本：v0.1  
日期：2026-06-26  
定位：闭源商用主程序 + 独立开源助手组件

## 1. 产品概述

本产品是一款面向私域运营、客服销售、门店导购、跨境/本地商家等场景的 Windows 微信多开工作台。产品提供账号集中管理、微信窗口承载、快捷话术、截图辅助、配置同步、授权会员等能力，帮助用户在一个桌面壳内管理多个微信会话并提升回复效率。

产品不直接改造微信客户端，也不承诺突破微信官方限制。涉及多开、防撤回等底层能力时，采用独立开源助手组件隔离分发，闭源主程序仅负责调用、状态展示和用户体验编排。

## 2. 背景与机会

目标用户常见痛点：

- 多个微信号需要同时登录，窗口分散，切换成本高。
- 常用回复、欢迎语、报价、售后解释等话术重复输入。
- 客服和销售人员需要截图、复制、标记、留档，但工具分散。
- 小团队需要统一配置话术库、账号备注和员工授权。
- 现有免费工具功能单一，缺少可持续更新、售后和商业化授权体系。

机会点：

- 以“多开 + 话术 + 窗口工作台”为第一价值闭环，不把产品只做成补丁工具。
- 商业化能力放在闭源主程序和云端服务，底层 GPL 能力以开源助手组件合规隔离。
- 用稳定、易用、低学习成本的工作台体验争取中小商家和私域团队付费。

## 3. 目标用户

核心用户：

- 私域运营人员：需要同时处理多个微信账号和高频回复。
- 中小商家客服：需要固定话术、售前售后模板、截图留证。
- 门店导购/销售：需要欢迎语、商品推荐、预约提醒、客户备注。
- 小型团队管理员：需要统一购买授权、管理员工设备和话术配置。

非目标用户：

- 需要大规模自动化群发、批量加好友、机器人营销的灰产用户。
- 需要绕过平台风控或进行账号异常操作的用户。
- 需要移动端微信控制能力的用户。

## 4. 产品目标

MVP 目标：

- 支持 Windows 下多微信启动和窗口集中管理。
- 支持账号列表、登录状态、窗口聚焦、关闭、拆分。
- 支持本地快捷话术库，包括分类、搜索、插入、导入导出。
- 支持基础截图工具和截图时隐藏当前窗口选项。
- 支持本地授权试用、设备绑定、会员状态展示。
- 采用 GPL 助手组件隔离架构，闭源主程序不复制、不链接 GPL 源码。

商业目标：

- 第一阶段验证个人版/专业版付费转化。
- 第二阶段提供团队版：席位、设备管理、统一话术配置。
- 建立可更新的开源助手组件分发和合规声明机制。

## 5. 范围定义

### 5.1 MVP 包含

桌面主程序：

- 顶部栏：品牌、版本、下载手机版入口、会员入口、收起侧栏、窗口控制。
- 左侧栏：账号头像占位、登录状态、添加账号、隐私锁、拆分窗口。
- 中间区：微信窗口承载区，支持登录二维码窗口定位和已登录窗口聚焦。
- 右侧栏：快捷回复搜索、分类 tab、话术列表、短语按钮、编辑入口、导入导出。
- 底部工具：截图工具、截图时隐藏当前窗口开关。

账号与窗口：

- 添加微信账号入口。
- 检测本机微信进程数量。
- 启动新的微信实例。
- 聚焦指定微信窗口。
- 关闭指定窗口或全部窗口。
- 拆分窗口到独立桌面窗口。
- 隐私锁：隐藏会话区域并要求本地密码或系统认证解锁。

快捷话术：

- 分类：文本、表情、文件。
- 默认分类：常用、欢迎、商品推荐、用户安抚、邀约好评、结束语、收藏。
- 话术字段：标题、正文、分类、标签、排序、更新时间。
- 支持搜索、点击复制、双击发送到当前输入焦点。
- 支持 JSON/CSV 导入导出。
- 支持底部常用短语按钮。

授权与更新：

- 设备指纹生成。
- 试用期状态展示。
- 激活码或账号登录授权。
- 会员等级展示。
- 主程序自动检查更新。
- 开源助手组件版本检查、下载和校验。

合规与声明：

- 设置页展示第三方开源声明。
- 展示 RevokeMsgPatcher / GPLv3 来源、许可证和源码地址。
- 主程序与 GPL 助手组件独立分发、独立进程运行。

### 5.2 MVP 不包含

- 自动群发、批量加好友、自动养号。
- 破解微信协议、读取或修改微信数据库。
- 云端同步聊天记录。
- 自动回复机器人。
- 手机端远程控制。
- 直接内置或静态链接 GPL 源码到闭源主程序。

## 6. 核心用户流程

### 6.1 首次启动

1. 用户安装并启动主程序。
2. 主程序检查运行环境、管理员权限和微信安装路径。
3. 主程序检查开源助手组件是否存在。
4. 如不存在，提示下载助手组件并展示 GPLv3 开源声明。
5. 用户确认后下载助手组件，主程序校验版本和哈希。
6. 用户进入主界面，开始添加微信账号。

### 6.2 添加账号

1. 用户点击“添加账号”。
2. 主程序调用助手组件尝试启动新微信实例。
3. 中间区显示微信登录窗口或二维码窗口。
4. 用户扫码登录。
5. 主程序识别窗口标题、进程 ID 和登录状态，生成账号卡片。
6. 用户可设置账号备注和头像。

### 6.3 使用快捷话术

1. 用户点击某个微信窗口，使其成为当前会话目标。
2. 在右侧搜索话术或切换分类。
3. 单击话术复制到剪贴板，双击尝试粘贴到当前输入框。
4. 用户手动确认发送。
5. 常用短语按钮支持快速复制或粘贴。

### 6.4 截图

1. 用户点击“截图工具”。
2. 若开启“截图时隐藏当前窗口”，主程序先隐藏自身窗口。
3. 调用系统截图能力或内置截图模块。
4. 截图完成后恢复窗口。
5. 图片进入剪贴板，用户可粘贴到微信。

## 7. 功能需求

### 7.1 主界面布局

- 主窗口默认尺寸建议为 1260 x 800。
- 左侧账号栏宽度约 88 px。
- 右侧话术栏宽度约 280 px。
- 中间区域自适应剩余宽度。
- 支持最小化、最大化、关闭。
- 支持收起右侧话术栏。

验收标准：

- 在 1366 x 768、1920 x 1080 下布局不重叠。
- 账号栏、微信承载区、话术栏三列清晰。
- 右侧话术列表长文本显示省略号，不撑破布局。

### 7.2 微信窗口管理

- 支持检测 `WeChat` 进程。
- 支持关闭微信单实例互斥句柄以允许多开。
- 支持启动新微信进程。
- 支持枚举微信主窗口句柄。
- 支持窗口嵌入主程序承载区，或使用“非嵌入聚焦模式”作为降级方案。
- 支持窗口失效、进程退出后的状态刷新。

验收标准：

- 同一台机器可启动多个微信登录窗口。
- 关闭一个微信进程后账号列表状态能在 5 秒内更新。
- 当嵌入失败时，能提示并切换到独立窗口聚焦模式。

### 7.3 账号管理

- 每个账号记录：本地 ID、备注、头像路径、窗口句柄、进程 ID、登录状态、最后活跃时间。
- 支持添加、删除本地账号记录。
- 支持账号排序。
- 支持账号备注编辑。
- 支持一键关闭所有微信窗口。

验收标准：

- 程序重启后保留账号备注和排序。
- 进程不存在时账号显示为离线或待登录。

### 7.4 话术库

- 本地存储默认使用 SQLite。
- 支持分类、标签、搜索、排序。
- 支持 JSON/CSV 导入导出。
- 支持话术编辑弹窗。
- 支持快捷短语按钮。

验收标准：

- 搜索响应时间在 1000 条话术内小于 200 ms。
- 导入时能识别重复标题并提供覆盖/跳过策略。
- 导出文件能被再次导入。

### 7.5 授权系统

- 支持本地试用状态。
- 支持激活码授权。
- 支持设备绑定。
- 支持会员等级展示。
- 支持离线宽限期。

验收标准：

- 未授权用户可进入试用，但高级功能受限。
- 授权过期后仍保留本地数据，但限制新增账号或高级话术功能。
- 网络异常时显示明确状态，不影响已登录微信窗口的基本使用。

### 7.6 开源助手组件

职责：

- 多开相关底层操作。
- 可选的防撤回补丁操作。
- 提供命令行接口或本地 IPC。
- 独立版本号、独立许可证声明、独立源码地址。

主程序调用方式：

```text
helper.exe multi-instance start
helper.exe multi-instance close-mutex --pid <pid>
helper.exe patch status --app wechat
helper.exe patch apply --app wechat --path <path>
helper.exe version --json
```

约束：

- 闭源主程序不复制 GPL 源码。
- 闭源主程序不静态链接或动态链接 GPL 库。
- 助手组件作为独立可执行文件分发。
- 安装包和官网提供助手组件源码获取方式。

验收标准：

- 主程序删除助手组件后仍可启动，只是相关底层功能不可用。
- 用户能在设置页看到开源组件许可证和源码链接。
- 助手组件可独立运行并输出版本信息。

## 8. 非功能需求

性能：

- 主程序冷启动小于 3 秒。
- 账号列表刷新不阻塞 UI。
- 话术搜索在 1000 条数据内小于 200 ms。

稳定性：

- 微信窗口崩溃或退出不导致主程序崩溃。
- 助手组件调用失败时返回明确错误。
- 所有外部进程调用有超时控制。

安全：

- 不采集聊天记录。
- 不上传微信账号、联系人、会话内容。
- 授权系统只上传设备授权所需的最小数据。
- 本地配置和话术数据库可选择加密。

合规：

- 明确展示第三方开源许可证。
- 不宣传“破解”“绕过风控”“批量营销”等用途。
- 不提供自动化骚扰、群发或违规营销能力。

## 11. 关键风险

许可证风险：

- 风险：GPL 源码并入闭源主程序导致闭源不可行。
- 控制：开源助手组件独立仓库、独立进程、独立许可证声明。

平台风险：

- 风险：微信更新导致多开或窗口管理失效。
- 控制：助手组件独立更新，主程序支持降级模式。

安全风险：

- 风险：被用户误解为聊天数据采集工具。
- 控制：默认不读取聊天记录，隐私说明明确写入产品和官网。

商业风险：

- 风险：用户只需要免费多开，不愿为工作台付费。
- 控制：付费点放在话术效率、团队配置、窗口体验和售后更新。

## 12. 成功指标

产品指标：

- 首次启动到第一个微信登录窗口出现小于 30 秒。
- 新用户 10 分钟内完成至少 1 个账号登录和 1 条话术使用。
- 话术功能 7 日留存用户使用率大于 40%。

商业指标：

- 试用到付费转化率第一阶段目标 3%-8%。
- 付费用户 30 日退款率低于 10%。
- 团队版客户占月收入比例逐步提升到 30% 以上。
